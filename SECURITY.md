# 安全策略

## 支持范围

SoundPort 目前处于早期开发阶段。安全修复优先应用于 `main` 分支的最新版本；
尚未承诺为旧提交或非官方构建提供长期安全支持。

## 报告漏洞

请优先使用 GitHub 仓库的 **Private vulnerability reporting** 功能私密报告漏洞。
仓库所有者应在发布仓库后于：

`Settings → Security → Code security and analysis → Private vulnerability reporting`

启用该功能。

报告请包含：

- 受影响版本或提交
- 漏洞影响和必要前提
- 最小复现步骤或概念验证
- 建议修复方式（如有）

不要在公开 Issue、Discussion、日志或截图中披露漏洞细节、蓝牙设备地址、用户路径、
凭据或其他隐私数据。若私密漏洞报告尚未启用，请先通过不含敏感细节的方式联系仓库
所有者，请求建立私密沟通渠道。
